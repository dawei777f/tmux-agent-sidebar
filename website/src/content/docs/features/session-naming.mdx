---
title: Session naming (local LLM)
description: Auto-generate a one-word session label via any OpenAI-compatible local LLM.
---

The sidebar can label each agent session with a one-word title generated by a local LLM (Ollama, LM Studio, llama.cpp server, …) or any other OpenAI-compatible chat-completions endpoint. The title replaces the agent-type label in the sidebar column, keeping the narrow layout readable even when many agents are running at once.

The feature is **opt-in** — disabled unless both `@sidebar_llm_endpoint` and `@sidebar_llm_model` are set.

## Why

- Claude Code's own session name is often just the truncated first prompt, which isn't very informative.
- Codex and OpenCode panes currently fall back to the agent-type label (`codex`, `opencode`), so multiple Codex sessions are indistinguishable.
- An LLM-generated one-word title (think `refactor`, `deploy`, `docs`, `調査`) makes scanning a tall sidebar dramatically faster.

## Minimal Ollama setup

```tmux
set -g @sidebar_llm_endpoint "http://localhost:11434/v1/chat/completions"
set -g @sidebar_llm_model    "llama3.2:3b"
set -g @sidebar_llm_auto_rename "on"
```

Start Ollama, complete any task in a Claude / Codex / OpenCode pane, and within ~10 seconds the sidebar row updates with a generated title.

## When it fires

- **Auto** — on the first `Stop` hook of each session, matching how ChatGPT and claude.ai generate titles after the first assistant response. Requires `@sidebar_llm_auto_rename "on"`. Fires exactly once per session (the result is persisted on disk, so subsequent Stop events don't regenerate).
- **Manual** — focus a row in the sidebar and press **`Shift+R`** to force regeneration. Useful if you want to retry after changing models or after a noisy first attempt.

## Precedence

A Claude-set `/rename` label always wins over the LLM-generated title. This makes the feature purely additive — it fills in for sessions that have no user-chosen name (including every Codex and OpenCode pane, which have no user-rename mechanism today) and steps aside the moment you express explicit intent.

## Output constraints

The system prompt asks the model for a single short word (max 16 characters, matching the user's language). Small models occasionally ignore part of that instruction, so the response is post-processed defensively:

1. Surrounding quotes, backticks, and brackets are stripped.
2. The first whitespace-delimited token is taken.
3. Trailing punctuation (periods, commas, question marks) is trimmed.
4. The result is truncated to 16 Unicode scalar values.

## Options

| Option | Default | Description |
| --- | --- | --- |
| `@sidebar_llm_endpoint` | _(required)_ | Full `http://` URL of an OpenAI-compatible `/v1/chat/completions` endpoint. Plaintext only — TLS is not supported yet. |
| `@sidebar_llm_model` | _(required)_ | Model name sent in the request body (`llama3.2:3b`, `gpt-oss:20b`, `gpt-4o-mini`, …). |
| `@sidebar_llm_api_key` | _(empty)_ | Optional bearer token. When set, the sidebar sends `Authorization: Bearer <value>`. |
| `@sidebar_llm_auto_rename` | `off` | Set to `on` to generate a title automatically on the first `Stop` hook. |
| `@sidebar_llm_timeout_ms` | `15000` | HTTP connect + read/write timeout per request. |

## Storage and lifecycle

Generated titles are cached per session at `/tmp/tmux-agent-sidebar-names/<session_id>.txt`. The 10-second session-name poller reads this directory alongside the Claude session JSON so the TUI picks up new titles without any extra plumbing. The files survive only as long as `/tmp` does — which lines up with how the rest of the sidebar behaves (activity logs and tmux options all die with the tmux server). Manual regeneration overwrites the file in place.

LLM call errors are appended to `/tmp/tmux-agent-sidebar-llm.log` and never block the agent: the Stop hook spawns the rename subprocess fire-and-forget, and the `Shift+R` keybind does the same.

## Compatibility matrix

| Agent | Auto-rename | Manual (`Shift+R`) |
| --- | --- | --- |
| Claude Code | ✓ | ✓ |
| Codex | ✓ | ✓ |
| OpenCode | ✓ | ✓ |

All three adapters report a `session_id` on their `Stop` hook, which is the key the generated title is stored under.

## Limitations

- **Plaintext HTTP only** in v1. Remote endpoints that require HTTPS are not supported yet — wrap them locally (e.g. with `socat` or a reverse proxy) or use a local runner.
- **Prompt injection** via activity-log content is accepted as a known limitation. The output is a display string and is never executed.
- **Small-model quirks** — 3B-class models occasionally produce off-language or verbose output. Defensive post-processing keeps it short; `Shift+R` regenerates if the first attempt is poor.
